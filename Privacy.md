# Darios strict Privacy Policy

Dario AI is designed to run as many features as possible locally on the user's computer.

Protecting the user's privacy is an important part of Dario AI. The program is intended to help the user without collecting unnecessary personal data, sharing private information, or using data for advertising purposes.

## Basic Principle

Dario AI does not collect data for advertising, tracking, or any other commercial profiling purposes.

Everything you entrust to Dario should generally stay on your own computer. Conversations, notes, settings, reminders, and saved information are processed and stored locally whenever the used feature runs locally.

I, as the developer, do not have access to what you discuss with Dario, which commands you use, or what information you entrust to Dario.

## Local Storage

Dario AI may store certain data locally on your computer so that its features can work correctly.

This may include:

* settings
* saved notes
* reminders
* local conversation history
* status data
* configuration files
* temporary files for specific features

These files are stored on your own device and are not automatically sent to the developer.

## Microphone and Voice Input

Dario AI may use your microphone to recognize voice commands.

Microphone input is only used when a feature that requires voice input is active. Dario AI is not intended to permanently analyze private conversations or use voice data for advertising.

Depending on the speech recognition system used, some voice input may be sent to an external service if speech recognition does not run fully locally. This only applies to features where such a service is technically required.

If a fully local speech recognition system is used, voice input remains on your computer.

## Screenshots and Screen Recognition

Depending on the version and enabled features, Dario AI may use screenshots to analyze your screen.

Screenshots are only used for the active screen recognition feature. They are not intended to be permanently stored or sent to the developer.

If a local AI model is used for screen recognition, the processing remains on your computer.

If certain features use external services, data may technically be transmitted to those services. This depends on which feature is enabled and how Dario AI is configured.

## Commands and Conversations

Commands, questions, and answers are only used for the active assistant features.

Dario AI may use this information to:

* understand your request
* generate an answer
* execute a command
* save a reminder
* create a note
* temporarily keep conversation context

The developer does not receive access to your private conversations with Dario AI.

## No Advertising Data

Dario AI does not collect data for:

* personalized advertising
* tracking
* selling data to third parties
* marketing profiles
* analyzing user behavior for advertising

Dario AI is not an advertising program and not a tracking tool.

## Internet Connection

Some features may require an internet connection, for example:

* model downloads
* updates
* online requests
* external speech recognition
* external text-to-speech services
* certain AI features if they do not run locally

If a feature works fully locally, no external data transfer is required for that feature.

## External Services

Depending on the configuration, Dario AI may use external services.

These may include services for:

* speech recognition
* text-to-speech
* model downloads
* online requests
* AI-related functions

External services have their own privacy policies. Dario AI cannot fully control how third-party services process data on their systems.

## Developer Access

The developer of Dario AI does not have direct access to:

* your conversations with Dario
* your microphone input
* your screenshots
* your local notes
* your reminders
* your private files
* your locally saved data

Dario AI does not automatically send this data to the developer.

## User Responsibility

The user is responsible for what information they share with Dario AI.

You should avoid entering highly sensitive information if you are not sure whether the used feature runs fully locally.

This includes, for example:

* passwords
* private login credentials
* banking information
* secret documents
* confidential personal information
* private information about other people

## Temporary Files

Dario AI may create temporary files so that certain features can work properly.

These may include:

* screenshots
* cache files
* temporary status files
* temporary analysis files

Such files are only intended to be used for the active feature and may be deleted after use.

## Security

Dario AI is designed to work as safely and locally as possible. However, no software can guarantee absolute security.

Users should only download Dario AI from official sources and should not use modified versions from unknown websites.

## Changes to This Privacy Policy

This privacy policy may be updated in future versions if features, technical processes, or used services change.

The latest version of this file should always be available in the official Dario AI repository.

## Summary

Dario AI is designed to work as locally as possible, does not collect advertising data, and does not give the developer access to private conversations.

Microphone input, screenshots, and commands are only used for active assistant features.

The user keeps control over their data and should use external features consciously.
